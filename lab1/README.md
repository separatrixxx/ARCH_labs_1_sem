# Лабораторная работа 1: Документирование архитектуры в Structurizr

## Вариант 4: Сайт заказа услуг (profi.ru)

### Описание системы

Система представляет собой платформу для заказа услуг, аналогичную profi.ru.
Позволяет клиентам находить и заказывать услуги, а исполнителям — публиковать и управлять своими предложениями.

### Стек технологий

- **Frontend**: React, Next.js
- **Backend**: C++, userver
- **База данных**: PostgreSQL

### Данные предметной области

- **Пользователь** — зарегистрированный пользователь системы (клиент или исполнитель). Поля: имя, фамилия, логин
- **Услуга** — предложение исполнителя. Поля: название, цена
- **Заказ** — связь клиента с услугой и исполнителем. Поля: id услуги, id клиента, id исполнителя, статус

### Реализуемое API

| # | Метод | Описание |
|---|-------|----------|
| 1 | `POST /users` | Создание нового пользователя |
| 2 | `GET /users?login={login}` | Поиск пользователя по логину |
| 3 | `GET /users?name={mask}` | Поиск пользователя по маске имени и фамилии |
| 4 | `POST /services` | Создание услуги |
| 5 | `GET /services` | Получение списка услуг |
| 6 | `GET /services/{serviceId}` | Получение услуги по идентификатору |
| 7 | `POST /orders?serviceId={serviceId}` | Создание заказа с выбранной услугой |
| 8 | `GET /orders?userId={userId}` | Получение заказов пользователя |

### Роли пользователей

- **Клиент** — регистрация, авторизация, просмотр каталога услуг, оформление и оплата заказа
- **Исполнитель** — регистрация, авторизация, создание услуг, получение уведомлений о новых заказах

### Внешние системы

- **Платёжная система** — обработка оплаты при оформлении заказа
- **Email-сервис** — уведомление клиентов о новых услугах; уведомление клиента об успешной оплате заказа; уведомление исполнителя о новом заказе
